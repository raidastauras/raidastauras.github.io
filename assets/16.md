I built **advntyr**, an app that turns travel intent into an actual multi-day route. You describe what you want — say, *military museums, natural wonders, and hiking in Slovenia for 5 days* — and an AI agent builds the whole trip for you.

The agent has a set of tools and figures out on its own which to call and in what order. Some run server-side, some client-side. In practice that means: hammering Google Maps and other APIs for every interest you mentioned (noisy, inconsistent, requires a bunch of tricks just to get usable results), running multiple filters to clean out the junk and keep only what actually matches, enriching what's left with short descriptions, then clustering everything geographically, ranking by quality and density, and finally generating an optimized route for your time window. The map updates in real time as it works. What you see is already the result of a lot of LLM loops and sanity checks running under the hood.

Then you can keep refining through conversation — *remove all wine-related places*, *reorder the route to start from the north*, *add a rest day in the middle*. Each follow-up and the agent figures out what to do again.

The honest hard part isn't just implementing the tools — it's figuring out what tools are even needed in the first place. You have to think through what actually happens when a human plans a trip: every decision made, every lookup done, every tradeoff considered. Then figure out how to approximate that entire process as a set of tools an AI agent can actually use. Get it wrong and the agent either can't do what the user asked, or does it badly. That design work, no longer the code, is where most of the real effort goes.

This is how trip planning should work. And honestly how I'd enjoy doing it myself. Hope I can pull it off.
