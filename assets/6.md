I wanted to learn more about APIs and how websites work, so I just started building stuff.

One thing I made was nameSpy — an API that scores how famous someone is online. Bart Simpson gets `0.95665`, Evan You gets `0.4668`. How it works:

1. **Check multiple sources like Google, Wikipedia, Twitter, and Instagram for data associated with a given name.**
2. **Extract relevant data, analyze it, and generate a score based on the average population.**

Also built another API that guesses someone's likely profession from their name — data scraping, language translation, and ML for entity detection.

Eventually lost interest and shelved both. But picked up a lot along the way — API gateways, logging, docker, serverless. And learned some things the hard way:

1. scraping Linkedin's data is nearly impossible
2. but you can trick google and use their google-translate services unlimited for free
3. the only way to reliably scrape google and Instagram data is to use proxies
4. there are no free proxies, and most TOR exit nodes are blocked by Google
5. Twitter had an open API — the others, not so much (looking at you, FB/IG).
