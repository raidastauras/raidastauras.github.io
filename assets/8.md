Government software is mostly giant, overengineered, and expensive. I always thought: it doesn't have to be. When I got a chance to test that, I took it.

I had to build a learning management system — a site where people can learn specific topics, admins can create and edit content, track user progress, see stats, etc. Could've used something like Moodle. But those systems are **giant monoliths, not graceful and efficient microservices**. So I built my own lean version instead.

Turns out I was right. The system runs on just two small cloud VMs and a free-tier MongoDB instance — handling around 250–400 unique visitors and up to 10k requests per day without breaking a sweat. Kept the database structure tight and compute on the server side minimal. Small, cheap, works. Monoliths don't have to be the only option, even for governments..?
